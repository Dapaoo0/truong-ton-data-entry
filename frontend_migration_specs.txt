3. ĐẶC TẢ CHI TIẾT CÁC FORM NHẬP LIỆU (UI COMPONENTS)
================================================================================
*** QUY TẮC CHUNG CHO TẤT CẢ CÁC FORM NHẬP LIỆU:
- Mọi form đều áp dụng "Cơ chế Queue" (Hàng đợi lưu tạm): User điền các Input -> Bấm nút "Thêm vào Danh sách" (Chưa ghi DB) -> Dữ liệu trôi xuống một "Bảng đợi" phía bên dưới.
- Bảng đợi (Queue Data Grid) cho phép xem lại các items vừa điền. Cuối mỗi dòng có cột xóa dòng (để chỉnh sửa nếu nhập lỗi).
- Dưới cùng của bảng đợi sẽ có nút bấm "🚀 Lưu toàn bộ" (Submit Bulk API ghi vào CSDL).
- Vùng dưới cùng của mỗi tab là "Bảng Dữ liệu thật" đã lưu trên hệ thống, kèm theo logic 48 Giờ:
  + Mỗi dòng data nếu lấy (Current_Time - Created_at <= 48h): Hiển thị nút "✏️ Sửa" (Mở ra một popup/modal form điền sẵn data) và nút "🗑️ Xóa".
  + Nếu data > 48 tiếng: Ẩn 2 tính năng Sửa/Xóa. Chuyển thành text "🔒 Quá 48h".

3.1. FORM KHỞI TẠO LÔ TRỒNG
- Input "Tên Lô": Text input (Bắt buộc nhập). Frontend nên viết Auto uppercase chữ cái đầu.
- Trạng thái "Loại trồng": Dropdown / Select box (Options: Trồng mới, Trồng dặm, Để chồi, Khác).
- Input "Ngày trồng": Date Picker. (Mặc định set current date).
- Input "Tuần": Text Input (Dạng Disabled/Read-only). Giá trị hiển thị phải tự động được tính là "Số tuần trong năm (ISO Week Number)" sinh ra từ biến Ngày Trồng bên cạnh.
- Input "Số lượng trồng": Number Input. Yêu cầu validate (> 0). Bước nhảy (Step) 100.

3.2. FORM CẬP NHẬT TIẾN ĐỘ (Stage Logs)
- Dropdown "Chọn Lô": Filter option các Tên Lô của hệ thống.
- Radio Group "Giai đoạn": Hiển thị ngang ngang. 
  + Nếu Team BVTV đăng nhập: Mặc định chọn 1 option "Chích bắp"
  + Nếu NT1/NT2 đăng nhập: Mặc định tùy chọn là "Cắt bắp"
- Input "Màu dây": Text input (Ví dụ: Đỏ, Xanh lá...). Validate bắt buộc.
- Input "Ngày thực hiện": Date Picker.
- Input "Tuần": Text (Dạng Disabled tự động tính tuần).
- Input "Số lượng": Number. (Min = 1, Step = 1000).

3.3. FORM ĐO SIZE
- Dropdown "Chọn Lô".
- Input "Màu dây": Text input.
- Radio Group "Lần đo": Options (1 hoặc 2). 
  + VALIDATION ĐẶC BIỆT: Nếu bấm chọn "Lần 2", Submit form phải có action call check DB: "Trường hợp [Lô này] + [Màu dây này] chưa từng có record đo Lần 1 => Không cho submit, báo lỗi".
- Input "Ngày đo": Date Picker. Input "Tuần" (Auto).
- Input "Hàng kiểm tra": Text (VD: H1-H5).
- Input "Size (Cal)": Number Field (Kiểu số thập phân, ví dụ 12.5). Step 0.1.
- Input "Số lượng buồng mẫu": Number Field. Step 10.

3.4. FORM KIỂM KÊ CÂY (Tree Inventory Logs)
- Gồm: Dropdown chọn Lô, Ngày kiểm kê (Date), Tuần (Auto), và Số lượng cây thực tế (Number input).

3.5. FORM ĐO PH ĐẤT (Cấu trúc mảng Form động - Dynamic Repater UI)
- Phía trên quy định thông tin gốc: Dropdown Lô, Ngày Đo, Tuần Đo.
- Nửa dưới giao diện là các thẻ nhập động: Mặc định là 1 dòng "Lần đo 1" với một Text_Field nhập liệu (Kiểu dữ liệu Number từ 0 đến 14.0).
- Nút "➕ Thêm lần đo": Bấm vào Front-end sẽ đẻ thêm Element div dọc xuống dưới "Lần đo 2", "Lần đo 3"...
- Nút "Lưu kết quả": Quét duyệt mảng component array, bóc tách toàn bộ các Field value > 0 gửi 1 mẻ mảng JSON Payload POST lên server.
